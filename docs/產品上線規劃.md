# 產品上線規劃

Created: 2026-06-01 08:22:43 +08:00

FutureLight 的目標不是「基本可跑」，而是做到可正式上線、可被家長信任、可通過兒童產品審核、可長期營運的語言學習產品。任何實作、文件、Todo、測試與資源規劃，都要以本文件作為產品級標準。

## 產品定位

FutureLight 是給兒童使用、由家長管理的語言學習產品。第一版聚焦英文啟蒙，並以聽、說、看圖、遊戲化練習、家長追蹤為核心。

第一版承諾：

- 兒童區無廣告、無外部連結、無直接購買壓力。
- 家長區處理帳號、訂閱、資料管理與隱私同意。
- 孩子每 30-60 秒有一次互動，不只被動看影片。
- 所有課程內容需要可審核、可追溯、可版本化。
- 音效、影片、字幕、口說錄音都要有家長控制與可及性設定。

## 上線標準

### Gate 1 Product Spec Complete

- 完成目標年齡層：3-5、6-8、9-11。
- 完成競品矩陣與差異化定位。
- 完成頁面流程、按鈕導向、換頁動畫、點選音效規劃。
- 完成資料模型、API、PostgreSQL schema 初稿。
- 完成兒童隱私資料地圖。

### Gate 2 Alpha Web Complete

- Web 端完成首頁、孩子資料、課程探索、課程詳情、學習播放、練習遊戲、學習進度、獎勵成就、家長中心。
- React route 使用真實 URL，支援返回、深連結、未登入導向。
- Rust API 不只回硬編資料，需可讀取課程、步驟、進度、嘗試紀錄。
- PostgreSQL migration 可重建 local/staging DB。
- 每次交付跑 frontend build、Rust check/test、asset checker、content checker、browser smoke test。

### Gate 3 Privacy Review Ready

- 完成 COPPA / GDPR-K / Apple Kids Category / Google Play Families checklist。
- 完成家長同意、撤回同意、刪除孩子資料、資料匯出。
- 禁用兒童區第三方追蹤與行為廣告。
- 完成兒童版與家長版隱私政策。
- 完成 App Store privacy nutrition labels 與 Google Play Data Safety 草稿。

### Gate 4 Beta Ready

- TestFlight / Play Internal Testing 準備完成。
- 有測試帳號、審核說明、示範資料。
- 有錯誤追蹤、客服流程、內容錯誤回報流程。
- 有家長訪談與孩子可用性觀察記錄。

### Gate 5 Store Submission Ready

- 完成 App Store / Google Play metadata、截圖、預覽影片、關鍵字、副標題。
- 完成 age rating、content rating、Families policy、Kids category 申報。
- 完成訂閱、退款、取消訂閱說明。
- 後端與素材服務在審核期間可用。

### Gate 6 Launch Ready

- Production 不從本機 dev port 提供服務。
- 有 staging / production 分離、備份、監控、回滾。
- 有內容發布審核流程。
- 有營運儀表板：活躍孩子、完成率、留存、複習率、錯誤率、客服量。
- 有上線後 7 天、30 天、90 天迭代計畫。

## 功能完整度

### 兒童學習區

- 首頁：選孩子、繼續學習、今日複習、探索課程、獎勵入口。
- 課程探索：主題、程度、年齡、學習目標、離線狀態。
- 學習播放：看圖、聽音、跟讀、選答案、下一步。
- 練習遊戲：配對、聽音選圖、拼字、口說模仿。
- 進度：詞彙掌握、弱點、連續天數、完成單元。
- 獎勵：星星、徽章、貼紙簿、完成動畫。

### 家長與信任區

- 家長中心：孩子列表、學習時間、課程進度、弱點與建議。
- 設定：每日時間、難度、音量、字幕、麥克風、離線下載。
- 資料權利：匯出資料、刪除資料、撤回同意。
- 訂閱：只在家長區展示，必須通過家長閘門。

### 內容管理區

- 課程 CRUD、單元 CRUD、活動 CRUD。
- 素材選取與缺漏檢查。
- AI 素材記錄 prompt、模型、日期、用途、授權、審核者。
- 發布前必跑 `futurelight-assets` 與 `futurelight-content-checker`。

## 學習科學要求

- Spaced repetition：根據答題品質安排複習。
- Active recall：每段學習都要讓孩子主動回想或操作。
- Adaptive difficulty：成功率過低就降低難度，過高就提高挑戰。
- Immediate feedback：即時回饋，但避免羞辱、恐嚇或過度刺激。
- Multi-modal learning：圖像、聲音、動作、文字要互相支援。

## 不接受的狀態

- 只做一個 dashboard 或 landing page。
- 只有硬編假資料，沒有資料流設計。
- 沒有家長同意與資料刪除流程。
- 沒有音效、字幕、靜音、減少動畫規劃。
- 直接使用 `5173 / 4000 / 5433` 啟動 FutureLight。
- AI 生成內容直接進孩子區，沒有人工審核。
- App Store / Google Play metadata 到最後才補。
