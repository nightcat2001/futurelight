# 登入與註冊功能設計

## 功能 1：家長註冊

家長可使用 Email 與密碼建立帳號。

規則：

- Email 必須唯一。
- 密碼需符合最低安全強度。
- 註冊成功後導向建立孩子資料。
- 註冊失敗需顯示明確原因。

## 功能 2：登入

家長或管理員可使用 Email 與密碼登入。

規則：

- 登入成功後建立 session 或發放 token。
- 登入後回到原本要前往的頁面；若沒有來源頁，回首頁。
- 多次登入失敗需啟動頻率限制。

## 功能 3：登出

使用者可登出目前裝置。

規則：

- 登出後清除前端登入狀態。
- 後端 session 或 refresh token 必須失效。
- 登出後不可存取需要登入的頁面。

## 功能 4：忘記密碼

家長可輸入 Email 取得密碼重設連結。

規則：

- 不透露 Email 是否存在。
- 重設 token 需有過期時間。
- token 使用後不可再次使用。

## 功能 5：角色導向

登入後依角色導向不同首頁。

規則：

- 家長導向前台首頁。
- 管理員導向內容管理頁。
- 無權限角色不可進入管理後台。
