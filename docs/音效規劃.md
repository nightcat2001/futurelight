# 全站音效規劃

## 設計原則

音效用來支援孩子理解操作結果、保持學習節奏與提供鼓勵，不用來製造干擾。

原則：

- 音效要短，避免蓋過語音教學內容。
- 語音、發音、故事音檔的優先權高於 UI 音效。
- 所有音效都必須可關閉。
- 家長可在系統設定中調整音效與語音音量。
- 答錯音效不可刺耳，避免讓孩子有挫折感。

## 音量分類

| 類型 | 預設音量 | 說明 |
| --- | --- | --- |
| 教學語音 | 100% | 單字、句子、故事、聽力題音檔 |
| 操作音效 | 45% | 點擊、切換、開啟、關閉 |
| 回饋音效 | 60% | 答對、答錯、完成步驟 |
| 獎勵音效 | 70% | 星星、徽章、任務完成 |
| 背景音樂 | 預設關閉 | MVP 不建議預設開啟 |

## 音效資源格式

- 短音效：`webm` 優先，備援 `mp3`。
- 教學語音：`mp3` 或 `aac`，需兼顧瀏覽器支援。
- 單一 UI 音效建議小於 100KB。
- 教學語音需可被內容管理系統上傳與替換。

## 全站音效清單

| 音效 ID | 用途 | 觸發時機 |
| --- | --- | --- |
| `ui_click` | 一般點擊 | 主要按鈕點擊 |
| `ui_toggle` | 切換選項 | Tab、開關、篩選器切換 |
| `ui_error_soft` | 柔和錯誤 | 表單錯誤、操作失敗 |
| `learning_step_complete` | 學習步驟完成 | 點擊我學會了 |
| `learning_unit_complete` | 單元完成 | 完成最後學習步驟 |
| `answer_correct` | 答對 | 練習答對 |
| `answer_wrong_soft` | 答錯 | 練習答錯 |
| `reward_star` | 星星增加 | 獲得星星或點數 |
| `reward_badge` | 徽章取得 | 新徽章解鎖 |
| `mission_complete` | 任務完成 | 任務進度達標 |
| `page_success` | 儲存成功 | 設定、孩子資料、內容管理儲存成功 |

## 播放規則

- 同一時間只能有一個教學語音播放。
- 教學語音播放中，操作音效可以播放，但音量需降低或避免重疊。
- 答題回饋音效必須等使用者送出答案後播放。
- 獎勵音效不可在每次小操作重複播放，只在實際獲得獎勵時播放。
- 快速連點按鈕時，`ui_click` 需節流，避免連續疊音。

## 設定與無障礙

使用者可調整：

- 音效開關。
- 教學語音音量。
- UI 音效音量。
- 是否自動播放教學語音。

無障礙要求：

- 不能只靠音效傳達結果，畫面也要有文字或視覺回饋。
- 靜音模式下，所有流程仍可完整使用。
- 錯誤、成功、獎勵都需要有可視化提示。

## PostgreSQL 資料需求

可先用設定欄位支援 MVP：

- `user_preferences.sound_enabled`
- `user_preferences.voice_volume`
- `user_preferences.effect_volume`
- `user_preferences.auto_play_voice`

內容音檔沿用媒體資源：

- `media_assets.asset_type = audio`
- `media_assets.url`
- `media_assets.duration_ms`
- `media_assets.mime_type`

## React 實作方向

- 建立全站 `SoundProvider` 管理音效開關與音量。
- 建立 `playSound(soundId)` 統一播放 UI 音效。
- 教學語音與練習題音檔由頁面播放器控制，不混在 UI 音效工具中。
- 首次播放需符合瀏覽器自動播放限制，通常要等使用者互動後才能播放。

## Rust API 實作方向

- 使用設定 API 儲存音效偏好。
- 內容管理 API 支援音檔上傳、替換、刪除與引用檢查。
- 學習事件可記錄教學音檔播放，但不需要記錄每個 UI 點擊音效。
