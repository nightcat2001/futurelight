# 法規與上架檢查

Created: 2026-06-01 08:32:25 +08:00

本文件不是法律意見，而是產品與工程執行 checklist。FutureLight 只要面向兒童，就必須把 Apple、Google、FTC/COPPA、資料安全與家長控制當成核心功能，而不是上架前補文件。

## 1. Apple App Store

### 1.1 Kids Category

必做：

- 選擇正確 age band：5 and under、6-8、9-11。
- 孩子區不可有外部連結、購買機會或干擾，除非放在家長閘門後。
- Kids Category app 不應包含第三方 analytics 或第三方 advertising。
- 若使用有限第三方 analytics，不能收集 IDFA、孩子姓名、生日、email、位置、裝置或網路識別資訊。
- 必須有隱私政策。
- 必須遵守兒童隱私法規。

FutureLight 決策：

- 兒童區不載入廣告 SDK。
- 兒童區不載入一般 third-party analytics。
- 家長區才處理訂閱、外部政策連結、客服連結。
- 所有外部 URL 需通過家長閘門。

### 1.2 App Store Connect 準備欄位

- App name
- Subtitle
- Category
- Made for Kids checkbox
- Age rating questionnaire
- Privacy nutrition labels
- App Privacy policy URL
- Subscription products
- Review notes
- Demo account
- Screenshots：iPhone / iPad 尺寸
- App preview video
- Content rights declaration

## 2. Google Play

### 2.1 Target Audience and Content

必做：

- 在 Play Console 宣告 target age group。
- 只選真正設計支援的年齡層。
- 若任一 target audience 包含 children，需符合 Families Policy。
- 上架前需提供 app access instructions、privacy policy、ads declaration。
- Google 會審核宣告的目標族群是否與 app 內容一致。

FutureLight 決策：

- 年齡層在產品中明確分為 3-5、6-8、9-11。
- Play Console 宣告需與 UI、素材、商店文案一致。
- 若未做成人版，不宣告 all ages 只為擴大觸及。

### 2.2 Families data practices

禁止或嚴格限制：

- Android Advertising ID
- SIM serial
- Build serial
- BSSID / SSID / MAC
- IMEI / IMSI
- phone number

FutureLight 決策：

- Android build 不請求不必要裝置識別權限。
- 任何 SDK 導入前先填 SDK inventory。
- 兒童與 unknown age 預設不傳 advertising ID。

### 2.3 Google Play 上架資料

- Store listing
- Short description
- Full description
- Feature graphic
- Screenshots
- App category
- Content rating
- Target audience and content
- Data Safety
- Ads declaration
- Privacy Policy URL
- Families Policy review
- Test account
- In-app products / subscriptions

## 3. FTC / COPPA

COPPA 適用於 directed to children under 13 的網站、app 與 online services，也適用於一般服務但實際知道自己收集 13 歲以下兒童資料的情境。

必做：

- 發布清楚隱私政策。
- 向家長提供 direct notice。
- 收集兒童個資前取得 verifiable parental consent。
- 允許家長查看、刪除兒童資料。
- 限制資料保存期限。
- 只收集完成服務必要資料。
- 2025 更新方向強調第三方廣告/資料分享需要更明確 opt-in。

FutureLight 決策：

- 預設只收集 child nickname、age band、learning progress。
- 不收集真實生日，除非為法規合規且經家長同意。
- 口說錄音預設短期處理；保存需家長同意。
- 刪除請求必須刪除原始資料與衍生學習資料。

## 4. 隱私資料地圖

| 資料 | 來源 | 是否兒童資料 | 用途 | 保存 | 家長同意 | 刪除 |
| --- | --- | --- | --- | --- | --- | --- |
| Parent email | 家長註冊 | 否 | 登入、通知、同意紀錄 | 帳號存在期間 | 不需兒童同意 | 可刪 |
| Child nickname | 家長建立 | 是 | 顯示孩子 profile | profile 存在期間 | 需要 | 可刪 |
| Age band | 家長選擇 | 是 | 適齡內容 | profile 存在期間 | 需要 | 可刪 |
| Learning attempts | 孩子答題 | 是 | 進度與 SRS | 可設定保存期 | 需要 | 可刪 |
| Pronunciation score | 口說活動 | 是 | 即時回饋 | 儲存分數，不必存原音 | 需要 | 可刪 |
| Voice recording | 麥克風 | 是，敏感 | 口說評分 | 預設短期 | 明確需要 | 可刪 |
| Reward events | 學習流程 | 是 | 成就與動機 | profile 存在期間 | 需要 | 可刪 |
| Subscription status | 家長購買 | 否 | entitlement | 合約/稅務要求 | 成人操作 | 可依規則刪 |
| Audit logs | 系統 | 可能 | 安全與同意證明 | 法規/安全期限 | 視資料而定 | 依政策 |

## 5. 家長閘門規格

家長閘門必須保護：

- 訂閱 / 購買
- 外部連結
- 隱私政策 / 服務條款外開
- 麥克風設定
- 資料匯出
- 刪除孩子資料
- 切換到內容管理或家長中心

可選方式：

- 成人數學題
- email one-time code
- passcode
- 生物辨識由裝置系統處理

不可使用：

- 孩子容易誤觸的普通確認框。
- 只靠長按 3 秒。
- 只靠「你是家長嗎？」。

## 6. SDK Inventory

任何 SDK 導入前填表：

| SDK | 用途 | 兒童區載入 | 收集資料 | 第三方分享 | Apple Kids 可用 | Google Families 可用 | 決策 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| React | UI | 是 | 無 | 無 | 可 | 可 | 可用 |
| Vite | dev/build | 否 | 無 | 無 | N/A | N/A | 可用 |
| Sentry 類 | error tracking | 待定 | crash/device | 是 | 需審核 | 需審核 | 先不用於兒童區 |
| Analytics 類 | analytics | 否 | events | 是 | 不建議 | 需審核 | 不進兒童區 |
| Stripe | Web payment | 家長區 | payment | 是 | 家長區 | 家長區 | 只在家長區 |
| AI speech API | pronunciation | 待定 | audio | 是 | 需同意 | 需同意 | 需家長同意與短期保存 |

## 7. 上架前 Blocker

- 沒有 privacy policy URL。
- 沒有 data deletion 流程。
- 沒有家長閘門。
- SDK inventory 不完整。
- App Store privacy labels 與實際 SDK 不一致。
- Google Data Safety 與實際資料收集不一致。
- 兒童區出現外部連結或付款入口。
- 未審核 AI 圖像/影片/音訊出現在孩子區。
- 口說錄音未取得家長同意。
